Strava Club Events

Application Limits

To stay within Strava API rate limits and keep the app fast, certain limits are applied when fetching your club events. Here's what they are and how they affect you.

Clubs

Up to … clubs are fetched from Strava to determine how many clubs you belong to. Events are then processed for only the … most recent clubs (in the order returned by Strava). If you are a member of more than … clubs, events from the remaining clubs will not be shown.

A per-club filter is planned for a future release, which will let you choose which clubs to include in the calendar view.

Events per Club

A maximum of … upcoming events per club are fetched. If a club has more than … upcoming events scheduled, only the first … will be shown. In practice, most clubs will have far fewer upcoming events than this.

Route Details

Detailed route information (distance, elevation gain, estimated time, etc.) requires a separate API request per event. To avoid hitting Strava rate limits, route details are fetched for at most … events per page load. Events beyond this limit will show only basic route info available in the event data itself.

Route details are cached after the first fetch, so events you've seen before will have full route info on subsequent loads regardless of the limit.

Note

These limits are subject to change. If you are regularly hitting them and finding it affects your experience, please open an issue on the project repository.
